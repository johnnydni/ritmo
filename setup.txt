## Setup

## Supabase Setup (Auth + DB)

### 1. Projekt anlegen
1. Konto bei [supabase.com](https://supabase.com) erstellen (free tier reicht)
2. Neues Project erstellen → Region: Frankfurt (eu-central-1)
3. Settings → API: URL + anon-public key kopieren

### 2. Env-Vars setzen
```bash
cp .env.example .env.local
# .env.local mit deinen Werten editieren
```

`.env.local` (NICHT committen!):
```
VITE_SUPABASE_URL=https://xxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGc...
```

Für GitHub Actions: Repo Settings → Secrets → Actions:
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

### 3. Email Auth aktivieren
1. Supabase Dashboard → Authentication → Providers
2. **Email** ist by default aktiv
3. Settings: "Confirm email" aktiviert lassen für Verifikation
4. URL Configuration → Site URL: `https://<user>.github.io/ritmo-padel/`
5. Redirect URLs ergänzen: `https://<user>.github.io/ritmo-padel/**`

### 4. Google OAuth
1. [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → Credentials
2. OAuth 2.0 Client ID erstellen (Web application)
3. Authorized redirect URIs: `https://<your-project>.supabase.co/auth/v1/callback`
4. Client ID + Secret in Supabase → Auth → Providers → Google eintragen
5. Provider aktivieren

### 5. Datenbank-Sicherheit (Row Level Security)
Sobald du eigene Tabellen anlegst (z.B. für Tournaments):
```sql
ALTER TABLE tournaments ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users see own tournaments" ON tournaments
  FOR SELECT USING (auth.uid() = host_id);

CREATE POLICY "users create their own" ON tournaments
  FOR INSERT WITH CHECK (auth.uid() = host_id);
```

RLS macht: jeder User sieht nur eigene Daten — auch wenn anon-key public ist.

## GitHub Pages Deployment

1. Repo erstellen, Code pushen
2. Settings → Pages → Source: **GitHub Actions**
3. Settings → Secrets and variables → Actions → **Secrets**: `VITE_SUPABASE_URL` + `VITE_SUPABASE_ANON_KEY` setzen
4. Workflow läuft automatisch → live unter `https://<USER>.github.io/<REPO>/`

### Custom Domain (Apex z.B. `ritmopadel.app`)

Der Workflow erkennt einen Custom Domain automatisch über eine
Repository-**Variable** (nicht Secret) und schaltet dann `base` von
`/<repo>/` auf `/` um, schreibt eine `dist/CNAME`-Datei und deployt.

**1. Repo-Variable setzen**
- Settings → Secrets and variables → Actions → **Variables** → "New repository variable"
- Name: `CUSTOM_DOMAIN`
- Value: `ritmopadel.app` (ohne `https://`, ohne Pfad, ohne Trailing-Slash)

**2. DNS beim Domain-Provider konfigurieren**

Für Apex-Domain (`example.com`) — vier A-Records auf die GitHub-Pages-IPs:
```
@   A   185.199.108.153
@   A   185.199.109.153
@   A   185.199.110.153
@   A   185.199.111.153
```
Optional zusätzlich vier AAAA-Records (IPv6, robustness):
```
@   AAAA   2606:50c0:8000::153
@   AAAA   2606:50c0:8001::153
@   AAAA   2606:50c0:8002::153
@   AAAA   2606:50c0:8003::153
```
`www` Subdomain als zusätzlichen Redirect:
```
www   CNAME   <USER>.github.io.
```

Für Subdomain-only (`app.example.com`) reicht ein einzelner CNAME:
```
app   CNAME   <USER>.github.io.
```
(Punkt am Ende beachten — manche Provider verlangen ihn.)

**3. GitHub Pages Settings**
- Settings → Pages → **Custom domain**: `ritmopadel.app` eintragen → Save
- DNS-Check läuft automatisch (kann 10 min - 24 h dauern)
- Sobald grün: **Enforce HTTPS** aktivieren (Let's-Encrypt-Cert wird auto-provisioniert)

**4. Supabase Auth-URLs anpassen**
- Dashboard → Authentication → URL Configuration
- Site URL: `https://ritmopadel.app/`
- Redirect URLs ergänzen: `https://ritmopadel.app/**`
- (Alte `<user>.github.io`-Einträge können bleiben, schaden nicht)

**5. Google OAuth Authorized Redirect**
Falls Google-OAuth genutzt wird: Die Supabase-Callback-URL ändert sich
nicht (bleibt `https://<projekt>.supabase.co/auth/v1/callback`).

**Troubleshooting**
- Build deployt, aber Domain zeigt 404: Custom-Domain-Eintrag in GH Pages
  Settings prüfen — wurde evtl. von einem alten Deploy ohne `CNAME` Datei
  überschrieben. Variable `CUSTOM_DOMAIN` muss gesetzt sein.
- HTTPS-Cert-Fehler nach DNS-Switch: 10–30 min warten, danach
  "Unenforce HTTPS" → Save → "Enforce HTTPS" → Save (toggle).
- Workflow nutzt weiterhin `/<repo>/` Base: Variable nicht in *Variables*
  sondern in *Secrets* eingetragen. `vars.CUSTOM_DOMAIN` liest nur
  Variables.

## Sprachansage (Voice-MP3-Hosting)

Wenn `voiceOn` in den Settings aktiv ist, lädt die App per HTTP einzelne
MP3-Dateien für Score-Ansagen. Die Base-URL wird im Code per
`voiceBaseUrl`-State gesetzt (UI-Eingabe ist aktuell deaktiviert,
Profil-spezifisch).

App lädt `{voiceBaseUrl}/{key}.mp3`. Format: MP3, ~30–50 KB pro Datei.
Bei fehlender Datei wird der Aufruf still ignoriert.

**Punkte (Bo3) — 16 Score-Files**
- `s-0-0.mp3`, `s-0-15.mp3`, `s-0-30.mp3`, `s-0-40.mp3`
- `s-15-0.mp3`, `s-15-15.mp3`, `s-15-30.mp3`, `s-15-40.mp3`
- `s-30-0.mp3`, `s-30-15.mp3`, `s-30-30.mp3`, `s-30-40.mp3`
- `s-40-0.mp3`, `s-40-15.mp3`, `s-40-30.mp3` (40-40 ⇒ `einstand.mp3`)

**Spezial (3)**
- `einstand.mp3`, `vorteil-a.mp3`, `vorteil-b.mp3`

**Golden Point (1)**
- `golden-point.mp3`

**Events (6)**
- `spiel-a.mp3`, `spiel-b.mp3`
- `satz-a.mp3`, `satz-b.mp3`
- `match-a.mp3`, `match-b.mp3`

**Hosting:** GitHub Pages, Cloudinary, Bunny CDN, S3.
SoundCloud streamt **nicht** direkt und funktioniert daher nicht.

## Smart Ring / Presenter — Keyboard-Modus

Die App fängt Eingaben aus Bluetooth-Tasten-Geräten ab (Smart Ring,
Presenter-Fernbedienung). Damit das auf iOS Safari überhaupt
funktioniert, muss ein versteckter Input fokussiert sein — das macht
`KeyCapture` automatisch.

**Tastenbelegung Presenter:**
- `Page Up` → Punkt Team A · Doppelklick = Letzten Punkt rückgängig
- `Page Down` → Punkt Team B · Doppelklick = Spiel zurücksetzen

**Tastenbelegung Smart Ring:**
- `2` → Punkt Team A
- `4` → Punkt Team B
- `Leertaste` → Letzten Punkt rückgängig
- `1` → Spiel zurücksetzen (zweimal `1` innerhalb 1.5 s = Reset rückgängig)

**Wenn der Eingabe-Tester nichts zeigt:** Dein Gerät sendet keine
Tastatur-Events. Häufige Ursache: der Ring bedient die iOS-Zoom-Funktion
statt eines Keyboards. Prüfe in den iOS Bedienungshilfen die
Zoom-Einstellung oder schau im Manual deines Geräts nach einem
Keyboard-Modus.

**Setup Smart Ring:** Ring per Bluetooth koppeln, RITMO Padel öffnen,
drücken — funktioniert sofort, sobald der Match-Screen aktiv ist.
